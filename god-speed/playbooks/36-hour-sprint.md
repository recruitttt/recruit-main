# 36-Hour Sprint Playbook

## Goal

Build a reliable demo with enough polish to feel real.

## Hours 0-2: Scope Lock

Do:
- Define mission
- Define demo thesis
- Create Linear project
- Assign owners
- Create P0/P1 issues

Artifact:
- Operating brief
- Demo-critical path

## Hours 2-8: Skeleton

Do:
- Frontend shell
- Backend/API skeleton
- Data schema
- AI prompt skeleton
- Demo data

Artifact:
- First deployed link

## Hours 8-18: Core Flow

Do:
- Build central workflow
- Integrate core pieces
- Create status visibility
- Start demo plan

Artifact:
- Demo path v1

## Hours 18-26: Integration and Verification

Do:
- Verify P0s
- Fix broken states
- Add fallback data
- Generate pitch/devpost drafts
- Create screenshots

Artifact:
- Verified demo-critical path

## Hours 26-32: Polish

Do:
- UI polish
- Copy
- Demo timing
- Backup video
- Devpost screenshots

Artifact:
- Final demo route
- Pitch script

## Hours 32-36: Submission

Do:
- Freeze
- Rehearse
- Submit
- Mark all issues Demo Ready or Cut

Artifact:
- Submission package
