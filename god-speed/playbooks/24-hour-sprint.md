# 24-Hour Sprint Playbook

## Goal

Ship the smallest impressive demo.

## Hours 0-1: Scope Lock

Do:
- Create operating brief
- Define demo thesis
- Identify P0 demo-critical path
- Create initial Linear issues

Cut:
- Anything not required for demo thesis

Artifact:
- Operating brief v1
- P0 issue list

## Hours 1-6: Skeleton

Do:
- App shell
- Data shape
- Demo data
- First route/screen
- First AI/output flow if relevant

Artifact:
- First deployed or runnable version

## Hours 6-14: Core Demo

Do:
- Build central workflow
- Connect main screens
- Create fallback data
- Verify basic flow

Artifact:
- Demo path v1

## Hours 14-18: Verification

Do:
- Verify P0s
- Create backup screenshots
- Fix demo-breaking bugs
- Mark unstable work mock/cut

Artifact:
- Verification checklist
- Backup plan

## Hours 18-21: Polish

Do:
- UI polish
- Copy polish
- Devpost draft
- Pitch script

Artifact:
- Demo script v1
- Devpost notes

## Hours 21-24: Freeze and Submit

Do:
- Stop new features
- Rehearse
- Record fallback video
- Submit

Artifact:
- Final demo
- Final Devpost
