# Linear Conventions

## Statuses

| Status | Meaning | When To Use |
|---|---|---|
| Backlog | Not yet scoped | Idea exists but not ready |
| Scoped | Accepted into sprint scope | We think this matters |
| Ready | Ready to be picked up | Has enough context to start |
| In Progress | Someone is actively working | Owner has claimed it |
| Blocked | Cannot continue without help/decision | Needs unblock |
| Needs Review | Built but not verified | Needs teammate/QA check |
| Verified | Works against checklist | Tested successfully |
| Demo Ready | Safe to show in demo | Verified on demo path |
| Cut | Removed from scope | Will not be built this sprint |

## Labels

| Label | Meaning |
|---|---|
| frontend | UI/client work |
| backend | API/server work |
| ai | AI/prompt/model work |
| data | Data/schema/seed work |
| infra | Deployment/config/devops |
| design | UX/UI/design systems |
| qa | Testing/verification |
| pitch | Pitch/demo/submission work |
| docs | Documentation |
| demo-critical | Demo breaks without this |
| stretch | Nice-to-have |
| blocked | Needs unblock |
| bug | Broken behavior |
| polish | Visual/copy polish |
| mockable | Can be safely mocked |
| cuttable | Can be removed if time is low |
| risky | Has high implementation/demo risk |
| needs-decision | Requires a decision before progress |

## Priorities

| Priority | Meaning |
|---|---|
| P0 | Demo breaks without this |
| P1 | Core product value |
| P2 | Useful but not required |
| P3 | Stretch/polish |

## Required Issue Fields

Every issue should include What, Why, Done Means, Verification, Blocked By, Unlocks, Demo Impact, Risks, Backup Plan, and AI Context.
