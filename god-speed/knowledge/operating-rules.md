# Operating Rules

1. Linear is the source of truth.
2. Hackathon OS is the execution layer.
3. AI skills operate on structured project context.
4. Every issue needs Done Means, Verification, Backup Plan, and Demo Impact.
5. If blocked for 20+ minutes, report it.
6. If it is not demo-critical, it can be cut.
7. Do not claim features that only work as mocks.
8. Update docs after implementation.
9. Verified means tested on the actual demo path.
10. No hidden work.
11. No vague almost done.
12. Stop adding features when rescue mode begins.
13. Prefer a reliable demo over a broader broken product.
14. Every demo-critical feature needs a fallback.
15. Cut aggressively when time is low.
