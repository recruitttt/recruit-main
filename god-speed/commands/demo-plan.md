# /demo plan

## Purpose

Creates the safest demo path.

## When To Use

Use once enough features work to define the live story.

## Reads

- project source of truth
- verified features
- backup plans
- known risks

## Writes / Proposes

- demo plan
- fallback artifact list

## Output Format

Demo thesis, audience, 3-5 minute flow, live click path, fallback path, screenshots/videos needed, what not to claim, closing line, presenter notes.

## Prompt

Design the safest honest demo. Prefer one reliable story over many fragile features.
