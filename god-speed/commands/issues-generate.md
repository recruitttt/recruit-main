# /issues generate

## Purpose

Generates Linear-ready issues from the operating brief.

## When To Use

Use after the operating brief is accepted or refreshed.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- .hackathon-os/knowledge/linear-conventions.md
- templates/linear-issue.md

## Writes / Proposes

- .hackathon-os/issues/generated-issues.md

## Output Format

Each issue includes title, owner, priority, status, labels, feature area, demo critical yes/no, can mock yes/no, can cut yes/no, what, why, done means, build notes, verification, blocked by, unlocks, demo impact, risks, backup plan, AI context.

## Prompt

Generate Linear-ready issues optimized for a 2-4 person team in a 24-36 hour sprint.
