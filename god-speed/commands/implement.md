# /implement

## Purpose

Plans or implements a selected feature/issue.

## When To Use

Use after selecting a specific issue from Linear or generated issues.

## Reads

- selected issue
- .hackathon-os/knowledge/project-source-of-truth.md
- .hackathon-os/knowledge/architecture-snapshot.md
- relevant code/docs
- templates/verification-checklist.md

## Writes / Proposes

- implementation plan
- docs/status updates
- Linear-ready completion comment

## Output Format

Implementation plan, files likely changed, exact build steps, verification steps, docs to update, Linear-ready completion comment, remaining risks, next unlocked issue.

## Prompt

Plan or implement the selected issue. Leave project state cleaner than before: update or propose docs, verification, and status.
