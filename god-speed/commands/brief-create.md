# /brief create

## Purpose

Creates or updates the operating brief.

## When To Use

Use at kickoff, after major scope changes, or when the team is unclear about the demo thesis.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- .hackathon-os/knowledge/operating-rules.md
- templates/operating-brief.md

## Writes / Proposes

- updated project-source-of-truth
- operating brief

## Output Format

Mission, demo thesis, target user, scoped MVP, stretch features, cut list, team roles, known risks, timeline, demo-critical path, verification plan, backup plan, immediate next actions.

## Prompt

Produce a concise operating brief for a 2-4 person hackathon team. Mark TODOs instead of inventing facts.
