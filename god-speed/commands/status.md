# /status

## Purpose

Generates teammate-readable sprint status.

## When To Use

Use at handoff, after major work, or fixed check-ins.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- .hackathon-os/issues/generated-issues.md
- .hackathon-os/status/latest.md
- .hackathon-os/knowledge/decision-log.md

## Writes / Proposes

- .hackathon-os/status/latest.md
- recommended Linear status updates

## Output Format

Demo readiness score, done, in progress, blocked, cut candidates, risks, next best actions, help needed, recommended Linear updates.

## Prompt

Create crisp status. Separate verified work from unverified work. Avoid vague almost done.
