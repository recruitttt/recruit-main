# /devpost

## Purpose

Writes Devpost notes.

## When To Use

Use before submission or when screenshots and project story are ready.

## Reads

- project source of truth
- demo plan
- verified features
- team notes

## Writes / Proposes

- Devpost draft notes
- links/screenshots checklist

## Output Format

Project name, tagline, inspiration, what it does, how it was built, challenges, accomplishments, what we learned, what is next, links/screenshots checklist.

## Prompt

Draft Devpost notes from verified project facts. Keep claims accurate.
