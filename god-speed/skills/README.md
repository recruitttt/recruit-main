# Skill Index

Skills are short role files for keeping an AI assistant focused on one job. Use them when a task spans more than one prompt or when the teammate needs a stricter role than a single slash command.

| Skill | File | Best For |
|---|---|---|
| Brief | `brief.md` | Operating brief creation |
| Issues | `issues.md` | Linear-ready issue generation |
| Next | `next.md` | Next-task recommendation |
| Implement | `implement.md` | Scoped feature implementation |
| Docs | `docs.md` | Source-of-truth and status sync |
| Blocker | `blocker.md` | Actionable blocker reports |
| Verify | `verify.md` | Completion and demo-readiness checks |
| Scope | `scope.md` | Keep/cut/mock/defer decisions |
| Demo | `demo.md` | Safe demo path design |
| Pitch | `pitch.md` | Pitch script writing |
| Devpost | `devpost.md` | Devpost submission notes |
| Rescue | `rescue.md` | Final-hours rescue planning |
| Template | `template.md` | New templates or command patterns |

## How To Use

1. Choose the skill that matches the work.
2. Provide the skill file plus the current source of truth, status, and relevant issue.
3. Ask the assistant to follow the skill's output format and chaos prevention rule.
4. Review any proposed scope, status, or demo-claim changes before treating them as final.
