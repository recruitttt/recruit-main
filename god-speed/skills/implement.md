# Skill: Implement

## Purpose

Plans or builds a selected feature without losing verification, docs, or status.

## When To Use

Use after a teammate selects a specific issue.

## Reads

- selected issue
- .hackathon-os/knowledge/project-source-of-truth.md
- .hackathon-os/knowledge/architecture-snapshot.md
- templates/verification-checklist.md
- relevant code and docs

## Writes / Proposes

- implementation plan
- files changed
- docs/status updates
- Linear-ready completion comment

## Inputs Required

- selected issue
- owner
- acceptance criteria
- current blockers

## Output Format

Implementation plan, likely files, exact build steps, verification steps, docs to update, Linear-ready comment, remaining risks, and next unlocked issue.

## Review Required?

Yes for demo-critical or risky changes.

## Chaos Prevention Rule

Do not only code. Update or propose updates to docs, verification, and status before calling the issue done.

## Prompt

Act as the Implement skill. Keep scope tight, follow existing project patterns, verify against the demo path, and leave the project state clearer than before.
