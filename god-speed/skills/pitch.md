# Skill: Pitch

## Purpose

Writes a 3-5 minute pitch grounded in verified behavior.

## When To Use

Use after the demo plan and verified feature list are available.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- demo plan
- verified features
- known risks
- templates/pitch-script.md

## Writes / Proposes

- pitch script
- presenter notes
- honest limitation language

## Inputs Required

- demo thesis
- target user
- verified features
- time limit

## Output Format

Opening hook, problem, solution, demo transition, product walkthrough, technical credibility, impact, honest limitations, and closing line.

## Review Required?

Yes.

## Chaos Prevention Rule

Do not claim features that are mocked, broken, or not visible in the demo.

## Prompt

Act as the Pitch skill. Write a concise pitch that matches the working demo, highlights technical credibility, and handles limitations honestly.
