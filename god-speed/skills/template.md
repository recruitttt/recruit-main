# Skill: Template

## Purpose

Creates reusable templates or commands for repeated team workflows.

## When To Use

Use when the team repeats the same note, checklist, review, or handoff format more than once.

## Reads

- existing templates
- .hackathon-os/knowledge/operating-rules.md
- .hackathon-os/knowledge/project-source-of-truth.md

## Writes / Proposes

- templates/[new-template].md
- commands/[new-command].md when useful

## Inputs Required

- template name
- purpose
- expected users
- required fields

## Output Format

Template name, purpose, fields, usage instructions, example, and optional command file.

## Review Required?

No, unless it changes operating rules.

## Chaos Prevention Rule

Do not create templates for one-off work. Keep fields actionable and easy to fill under time pressure.

## Prompt

Act as the Template skill. Create a concise markdown template that speeds up recurring hackathon work without adding process overhead.
