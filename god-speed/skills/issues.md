# Skill: Issues

## Purpose

Turns the operating brief into Linear-ready issues for a 2-4 person team.

## When To Use

Use after the brief is accepted or when scope changes enough that the issue list is stale.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- .hackathon-os/knowledge/linear-conventions.md
- templates/linear-issue.md

## Writes / Proposes

- .hackathon-os/issues/generated-issues.md
- Linear issue titles, owners, priorities, labels, and dependencies

## Inputs Required

- operating brief
- team roles
- sprint length
- known blockers

## Output Format

For each issue include title, owner, priority, status, labels, feature area, demo critical, can mock, can cut, what, why, done means, build notes, verification, blocked by, unlocks, demo impact, risks, backup plan, and AI context.

## Review Required?

Yes.

## Chaos Prevention Rule

Do not create vague umbrella issues. Split work so one owner can make visible progress quickly.

## Prompt

Act as the Issues skill. Create a practical issue list optimized for parallel work, demo reliability, and Linear import. Separate P0 demo path work from pitch, QA, polish, and stretch work.
