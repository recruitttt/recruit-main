# Skill: Devpost

## Purpose

Prepares accurate Devpost notes for submission.

## When To Use

Use once the project story, demo plan, and verified features are clear.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- demo plan
- verified features
- team notes
- templates/devpost-notes.md

## Writes / Proposes

- Devpost draft
- links checklist
- screenshots checklist

## Inputs Required

- project name
- tagline
- tech stack
- verified features
- screenshots/videos

## Output Format

Project name, tagline, inspiration, what it does, how it was built, challenges, accomplishments, what we learned, what is next, links, and screenshots checklist.

## Review Required?

Yes.

## Chaos Prevention Rule

Keep claims aligned with verified features and clearly separate future work from what was built.

## Prompt

Act as the Devpost skill. Draft submission notes that are accurate, specific, and ready for final teammate review.
