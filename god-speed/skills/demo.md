# Skill: Demo

## Purpose

Designs the safest honest demo path.

## When To Use

Use once enough features work to form a 3-5 minute story.

## Reads

- .hackathon-os/knowledge/project-source-of-truth.md
- verified features
- backup plans
- known risks
- templates/demo-plan.md

## Writes / Proposes

- demo plan
- live click path
- fallback artifact list

## Inputs Required

- audience
- verified features
- time remaining
- presenter roles

## Output Format

Demo thesis, audience, timed flow, live click path, fallback path, screenshots/videos needed, what not to claim, closing line, and presenter notes.

## Review Required?

Yes.

## Chaos Prevention Rule

Only include features that work or have an honest fallback. Do not script around imaginary behavior.

## Prompt

Act as the Demo skill. Choose one reliable story, make the click path explicit, prepare fallback artifacts, and state what the team must not claim.
