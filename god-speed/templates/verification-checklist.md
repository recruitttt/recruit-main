# Verification Checklist

Feature:
Owner:
Date:

## Functional Verification

- [ ] Core action works
- [ ] Expected output appears
- [ ] Error state does not crash app
- [ ] Works more than once

## UI Verification

- [ ] Layout is not broken
- [ ] Copy is understandable
- [ ] Loading/empty/error states are acceptable
- [ ] Looks presentable for demo

## Data Verification

- [ ] Uses correct demo data
- [ ] No undefined/null placeholder garbage
- [ ] State persists or is clearly mocked
- [ ] Seed data is realistic

## Demo Verification

- [ ] Appears in demo path
- [ ] Presenter knows how to show it
- [ ] Can be shown in under 60 seconds
- [ ] Works on deployed link or recorded fallback

## Fallback Verification

- [ ] Screenshot exists
- [ ] Video exists if needed
- [ ] Script adjustment exists
- [ ] Team knows what not to claim

## Logging / Observability Verification

- [ ] Important actions are visible in logs/console/status
- [ ] Failures are understandable
- [ ] Debug info is enough for final-hours triage

## Final Verdict

- [ ] Not Done
- [ ] Needs Review
- [ ] Verified
- [ ] Demo Ready
